> 本地优先、隐私优先的 Markdown 知识库桌面应用。
> 所有笔记都是磁盘上的纯 Markdown 文件，外加本地 LLM 能力（智能建议、改写、RAG 问答）。

## 1. 界面概览

LMNotes 主窗口采用三栏布局：

```
┌──────────────┬────────────────────────┬──────────────┐
│   左侧栏      │       中间编辑区         │   右侧栏      │
│              │                        │              │
│  搜索框       │   Markdown 编辑器        │  建议中心     │
│  + 新建       │   (CodeMirror)          │  (LLM 建议)  │
│  📥 导入      │                        │              │
│  💬 Chat      │   ✏️ 编辑 / 👁 预览      │              │
│              │                        │              │
│  📁 文件      │                        │              │
│  (文件树)     │                        │              │
└──────────────┴────────────────────────┴──────────────┘
```

| 区域 | 说明 |
|---|---|
| **左侧栏** | 搜索框、新建/导入按钮、Chat 入口、搜索结果列表、底部可折叠的「📁 文件」文件树 |
| **中间编辑区** | Markdown 编辑器（CodeMirror）；未打开笔记时显示「选择左侧笔记或搜索」 |
| **右侧栏** | 「建议中心」——展示 LLM 自动生成的待审建议（摘要、标签） |
| **右上角浮动 ⚙** | 打开 Provider 设置 |

弹窗/抽屉（按需打开）：

- **快速捕获**弹窗（`Ctrl+N`）
- **Provider 设置**弹窗（`Ctrl+,` 或点 ⚙）
- **Chat with Vault** 抽屉（`Ctrl+J`）

## 2. 笔记管理

### 2.1 新建笔记

有三种方式：

1. **左侧栏「+ 新建」按钮** → 弹出「笔记标题：」输入框（默认「新笔记」）→ 在 `notes/` 下创建笔记。
2. **文件树中点文件夹的「＋」** → 在该文件夹内新建。
3. **右键文件夹 →「📄 新建笔记」** → 同上。

新建的笔记自动带上 frontmatter（含 `type`、`id`、`title`、`created`）并以 `# 标题` 开头，随即在编辑器打开。

> 文件名规则：取标题中的字母/数字/`-`/`_`（转小写，最多 30 字符）作为 slug，再拼日期。中文标题若无可用的 slug，则用 id 的一部分代替。

### 2.2 编辑与预览

- 编辑器基于 **CodeMirror 6**，支持 Markdown 语法高亮、自动换行、标准撤销/重做（`Ctrl+Z` / `Ctrl+Y`）。
- **自动保存**：改动后停顿 **800ms** 自动保存；标题栏的 `●` 表示有未保存改动。
- **编辑/预览切换**：工具栏按钮在 `✏️ 编辑` 与 `👁 预览` 间切换。预览模式用 `marked` 渲染正文（不显示 frontmatter），可与编辑分屏对照。
- 工具栏显示当前笔记的相对路径。
- 保存后会自动重新索引，并在后台异步重新生成 LLM 建议。

### 2.3 快速捕获（Quick Capture）

按 **`Ctrl+N`** 打开快速捕获弹窗：

- 占位文字：「快速记一条…（Esc 关闭，Ctrl+Enter 保存）」
- 行为：把内容追加到 **今天的日记** `notes/daily/YYYY-MM-DD.md`。
  - 若当日日记不存在，会自动创建（`type: daily`，带 `# YYYY-MM-DD` 标题）。
  - 然后追加一个带 `## HH:MM` 时间戳的小节。
- `Esc` 关闭；`Ctrl+Enter` 保存。

> 适合随手记录灵感，不必关心放在哪——都会进当天日记。

### 2.4 导入文件

点左侧栏 **「📥 导入」** 打开系统文件选择器，支持：

| 格式 | 处理方式 |
|---|---|
| `.md` / `.markdown` / `.txt` | 直接按 UTF-8 文本导入 |
| `.pdf` | 提取纯文本（尽力而为，会去掉空行） |
| `.docx` | 转为 Markdown（段落→文本，表格→Markdown 表格） |
| `.xlsx` / `.xls` | 取第一个工作表转为 Markdown 表格 |

导入后会写入 `notes/`，若原文没有 frontmatter 会自动补上（已有 frontmatter 则保留并补 `id`）。不支持的格式会提示：「不支持的格式: .xxx（支持: pdf, docx, xlsx, txt, md）」。

### 2.5 文件夹管理

- **新建文件夹**：文件树中点文件夹的 **「📁＋」**，或右键文件夹 →「📁 新建文件夹」→ 输入名称。
- 文件夹名取字母/数字/`-`/`_`/`/` 字符（最多 50 字符）。

### 2.6 删除笔记

- 文件树中点文件的 **「🗑」**，或右键文件 →「🗑 删除」。
- 会弹出确认：「确定删除 "xxx"？此操作不可撤销。」
- 确认后删除文件并从索引中清除。

### 2.7 移动与重命名

**移动**笔记或文件夹有两种方式：

1. **拖拽**：把文件/文件夹拖到目标文件夹上（拖动时高亮目标文件夹）。
2. **右键 →「✂️ 移动到…」**：弹出对话框，列出所有文件夹，点目标即移动。

**在文件管理器中打开**：右键 →「🖥 在文件管理器中打开」，会用系统文件管理器选中该文件（Windows: `explorer /select`；macOS: `open -R`；Linux: `xdg-open`）。

> ⚠️ **当前没有内置「重命名」功能**。如需改名，可在文件管理器中改，或移动后重建。文件监听器会自动把外部改动重新索引。
>
> ⚠️ 移动**不会**重写笔记内部的链接路径，移动后原有链接可能失效。

### 2.8 插入图片

- **粘贴图片**：在编辑器中 `Ctrl+V` 粘贴剪贴板里的图片。
- **拖入图片**：把图片文件拖到编辑器。

行为：

- 图片按 **SHA-256 哈希**存储到 `assets/img/<前2位hex>/<完整hash>.<扩展名>`，**自动去重**（相同图片不重复存）。
- 在光标处插入 Markdown 图片链接 `![文件名](/<相对路径>)`。

## 3. 搜索

左侧栏顶部搜索框，占位文字「搜索…（回车）」：

- 输入关键词后按 **回车** 触发搜索（最多 50 条结果）。
- 结果列表显示笔记标题（无标题则显示路径）和路径；点击即在编辑器打开。
- 引擎：**Tantivy BM25 全文检索** + jieba 中文分词，并用 SQLite 元数据富化（路径/标题）。

**中文支持**：是的，jieba 分词，所以多字中文词可被检索（如「知识图谱」可被「知识」搜到）。

> ⚠️ **已知限制**：jieba Default 模式不做重叠切分，搜索词若与索引词不对齐可能漏匹配。例如正文是「注意力」（被切成一个 token），搜索「注意」可能匹配不到。向量检索（用于 Chat）不受此限制。

> 注：侧边栏搜索是**纯全文**检索；向量检索只用于 Chat with Vault。

## 4. Chat with Vault（RAG 问答）

按 **`Ctrl+J`** 或点左侧栏「💬 Chat with Vault」打开抽屉。

- **多轮对话**：历史持久化在本地（SQLite `chat_history` 表），重开仍保留。可点「清空」清除。
- **流式回答**：答案逐 token 流式显示（`●` 表示生成中）。
- **引用**：回答中的 `[n]` 是可点击的引用，点击会跳转到对应笔记并关闭抽屉。
- 输入框占位：「问一个问题…（Enter 发送，Shift+Enter 换行）」。

**工作原理**（检索增强生成）：

1. 把你的问题向量化，做 **向量 KNN 召回**（top 2K）。
2. 同时做 **全文 BM25 召回**（top 2K）。
3. 用 **RRF（倒数排名融合）** 合并两路，取 top 5 片段。
4. 拼成带编号引用的上下文（约 6000 字预算，按段落截断）。
5. 把「系统提示 + 最近 20 条历史 + 当前问题」发给 Chat 模型（温度 0.4）。

系统提示要求助手：作为知识库助手、每点用 `[n]` 引用、仅依据上下文回答、无相关信息时回答「我的笔记中暂无相关信息」。

> 需要已配置可用的 LLM provider；否则功能不可用。发送前会过隐私护栏（见第 7 节）。

## 5. 建议中心（LLM 建议）

右侧栏「建议中心」展示**待审**建议（加载时读取，空态显示「暂无待审建议」）。

**自动生成**：每次保存/索引笔记后，后台自动调用 LLM 生成两类建议：

| 类型 | 内容 |
|---|---|
| **summary（摘要）** | 一句话（≤50 字）总结笔记核心内容 |
| **tag（标签）** | 3–5 个标签 |

同时，笔记正文会被向量化写入向量索引（供 Chat 检索，不可见）。

**接受 / 拒绝**：每张建议卡片有 `✓`（接受，快捷键 Enter）和 `✕`（拒绝）：

- 点 `✓` → 标记为 `accepted` 并从列表移除。
- 点 `✕` → 标记为 `rejected` 并移除。

> ⚠️ **当前限制**：接受建议**仅标记状态**，暂未把摘要/标签写回笔记 frontmatter。「链接建议」类型在界面已渲染，但当前版本尚未实际生成。

## 6. 选段改写

在编辑器中**选中一段文本后右键**，弹出改写菜单（无选区时不出现）：

| 菜单项 | 作用 |
|---|---|
| **润色** | 保持原意，使文本更流畅专业 |
| **扩写** | 补充细节与例证 |
| **翻译为英文** | 翻译成英文 |
| **总结要点** | 用要点列表总结 |

执行时显示「改写中…」并禁用按钮。

- **改写前会自动快照**整篇文档（存到 `.lmnotes/llm/snapshots/`）作为安全网。
- LLM 结果**原地替换**选区，并记入编辑器历史。
- 路由：使用 **Rewrite** 任务对应的 provider。改写是用户主动操作，故不检查笔记的 `llm_local_only` 标记（但仍受全局护栏约束）。
- **撤销**：会话内用 `Ctrl+Z`；跨会话的快照在磁盘上（当前无浏览/恢复 UI）。

## 7. 设置：LLM Provider 与隐私护栏

点右上角 **⚙** 或按 `Ctrl+,` 打开「Provider 设置」（存于 `~/.lmnotes/config.json`）。

> ⚠️ **保存配置后需重启应用生效**。

### 7.1 Provider 配置

支持两类 provider：

| 类型 | 说明 |
|---|---|
| **Ollama（本地）** | id 固定 `ollama`，单实例。字段：Base URL、Chat Model、Embed Model、Embed Dim（默认 768） |
| **OpenAI 兼容** | 适配 OpenAI / GLM / 其它兼容 API。字段：id、Base URL、API Key、Chat Model、Embed Model、Embed Dim（默认 1024） |

**默认配置**：一个本地 Ollama（`http://localhost:11434`），chat 用 `qwen2.5:7b`，embed 用 `nomic-embed-text`，五类任务（summarize / link_suggest / embed / chat / rewrite）都路由到 Ollama。

### 7.2 健康状态

打开设置或点「重新探测」，会 ping 各 provider：

- ✓「可用」 / ✕「不可达」
- 启动时也会探测；若全部不可达，日志提示：「⚠ No healthy LLM provider…」。

### 7.3 隐私护栏

- **「允许云端 Provider（默认关闭，本地优先）」** 复选框（`guard.cloud_allowed`，默认 `false`）。
- `sensitive_patterns`：敏感词列表（仅能直接编辑 `config.json`，UI 未暴露）。内容命中敏感词时不会发给云端 provider。

护栏规则（`guard::check`）：

- **本地 provider（Ollama）始终放行**。
- **云端 provider** 仅在同时满足以下条件才放行：笔记未标 `llm_local_only`、未命中敏感词、且 `cloud_allowed` 为 true。
- 改写（Rewrite）豁免 `local_only` 检查。

### 7.4 单笔记隐私标记

在某笔记 frontmatter 加 `llm_local_only: true`，可标记该笔记**绝不**发往云端 LLM（摘要/标签生成时会跳过）。

## 9. 笔记格式（OKF）

所有笔记是带 **YAML frontmatter** 的 Markdown 文件（遵循 OKF v0.1）：

```markdown
---
type: note              # 必填
id: nt_20260628_1430_AB34   # 内部稳定 id（自动生成）
title: 我的笔记          # 推荐
created: 2026-06-28T14:30:00+08:00
tags: [AI, 笔记]
llm_local_only: false   # 可选：true 则不发云端 LLM
---

# 我的笔记

正文用 Markdown 书写。内部链接用 bundle 相对路径（以 / 开头）：
见 [相关笔记](/notes/related.md)
```

**frontmatter 字段**：`type`（必填）、`title`、`description`、`resource`、`tags`、`timestamp`、`id`、`aliases`、`status`、`language`、`created`，以及 `llm_local_only`。**未知字段会被原样保留**（往返安全）。

**链接**：只有以 `/` 开头的 bundle 相对链接（如 `[文本](/notes/b.md)`）才被视为内部图谱边；相对/外部链接被忽略。

**保留文件名**：`index.md`（目录索引）和 `log.md`（更新历史）是特殊文件，不被当作普通笔记。

## 10. 数据存储位置

| 内容 | 路径 |
|---|---|
| **vault 根**（笔记 + 资源） | `~/.lmnotes/default` |
| 笔记 | `~/.lmnotes/default/**/*.md` |
| 日记 | `~/.lmnotes/default/notes/daily/YYYY-MM-DD.md` |
| 图片 | `~/.lmnotes/default/assets/img/<2位hex>/<hash>.<ext>` |
| 应用内部数据 | `~/.lmnotes/default/.lmnotes/` |
| 索引数据库 | `~/.lmnotes/default/.lmnotes/index.sqlite` |
| Tantivy 全文索引 | `~/.lmnotes/default/.lmnotes/tantivy/` |
| 改写快照 | `~/.lmnotes/default/.lmnotes/llm/snapshots/` |
| LLM/MCP 配置 | `~/.lmnotes/config.json` |
| MCP 发现文件 | `~/.lmnotes/mcp.json` |

> Windows 上 `~` 即 `C:\Users\<你>`。

> ⚠️ vault 位置**当前固定**为 `~/.lmnotes/default`，尚无切换 UI。

## 11. 常见问题

**Q：LLM 功能都不工作？**
A：多半是没配置 provider 或 provider 不可达。打开设置看健康状态。最简单的方式是本地启动 Ollama 并保持默认配置（qwen2.5:7b / nomic-embed-text）。

**Q：搜索「注意」搜不到正文里的「注意力」？**
A：这是 jieba Default 分词模式的已知限制（token 不对齐则漏匹配）。换个完整词，或用 Chat（向量检索不受此限）。

**Q：接受建议后笔记没变化？**
A：当前「接受」仅标记状态，未回写 frontmatter。可手动把摘要/标签复制进笔记。

**Q：移动笔记后链接失效？**
A：移动操作不重写内部链接路径。建议用 bundle 绝对路径链接，或移动后手动修正。

**Q：外部工具改了笔记会怎样？**
A：LMNotes 的文件监听器会自动把外部对 `.md` 的改动重新索引、重新生成建议——所以用 VS Code/Git 改笔记完全没问题，搜索/向量会自动更新。

**Q：怎么把笔记暴露给 AI agent（如 Claude Desktop）？**
A：见 [MCP 接入指南](./mcp-agent-integration.md) 与 [MCP 接口说明](./mcp-api.md)。默认已内嵌一个只读 MCP server。
